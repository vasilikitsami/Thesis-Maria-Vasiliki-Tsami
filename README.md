# Οδηγός χρήσης n8n workflows

Αυτός ο φάκελος περιέχει ένα αρχείο JSON με όλα τα workflows που χρησιμοποιήθηκαν στο n8n για την υλοποίηση της διπλωματικής με τίτλο Ανάπτυξη Πιλοτικής Υπηρεσίας Υποστήριξης με Chatbot βασισμένο σε Μεγάλα Γλωσσικά Μοντέλα (LLMs).

Το αρχείο των workflows είναι το n8n_workflows.json

Το αρχείο αυτό μπορεί να γίνει import στο n8n, ώστε να εμφανιστούν όλα τα workflows στο περιβάλλον του χρήστη.

Τα workflows είναι αυτοματοποιημένες ροές εργασίας που έχουν δημιουργηθεί στο n8n.

Στο συγκεκριμένο σύστημα χρησιμοποιούνται για λειτουργίες όπως:

- συλλογή δεδομένων από αρχεία ή ιστοσελίδες,
- επεξεργασία και καθαρισμός κειμένου,
- δημιουργία αρχείων,
- αποθήκευση δεδομένων,
- δημιουργία embeddings,
- αποθήκευση σε vector database,
- ανάκτηση σχετικής πληροφορίας,
- χρήση LLM για παραγωγή απαντήσεων,
- αξιολόγηση απαντήσεων,
- αποθήκευση αποτελεσμάτων σε Google Sheets.

Με απλά λόγια, τα workflows περιγράφουν τα βήματα που ακολουθεί το σύστημα για να συλλέξει πληροφορία, να τη μετατρέψει σε κατάλληλη μορφή, να τη χρησιμοποιήσει σε chatbot και να αξιολογήσει τις απαντήσεις του.

Τα workflows δεν είναι έτοιμα να τρέξουν απευθείας σε οποιονδήποτε υπολογιστή.

Το αρχείο `n8n_workflows.json` περιέχει τη δομή των workflows, δηλαδή τα nodes και τη σύνδεση μεταξύ τους. Δεν περιέχει όμως τα προσωπικά credentials, τα API keys, τα tokens ή την πλήρη παραμετροποίηση του server.

Για να λειτουργήσουν, πρέπει πρώτα να εγκατασταθεί και να ρυθμιστεί το κατάλληλο περιβάλλον.

Αυτό σημαίνει ότι ο χρήστης πρέπει να έχει:

- εγκατεστημένο n8n,
- πρόσβαση σε Ollama,
- πρόσβαση σε Supabase,
- πρόσβαση σε Cohere Reranker,
- πρόσβαση σε Google Drive,
- πρόσβαση σε Google Sheets,
- τα σωστά credentials για όλες τις παραπάνω υπηρεσίες.

Χωρίς αυτά, τα workflows μπορούν να ανοιχτούν και να προβληθούν, αλλά δεν θα εκτελεστούν σωστά.

# Οδηγίες Εγκατάστασης πριν το import:

Πριν γίνει import το αρχείο JSON, πρέπει να υπάρχει εγκατεστημένο το n8n.

Το n8n μπορεί να τρέχει είτε:

- σε server,
- σε τοπικό υπολογιστή,
- μέσω Docker,
- ή σε άλλη self-hosted εγκατάσταση.

Στην παρούσα περίπτωση, τα workflows έχουν δημιουργηθεί για συγκεκριμένο server, τον spartacus του εργαστηρίου Euclid. Επομένως, για να εκτελεστούν σωστά σε νέο υπολογιστή, πρέπει να εγκατασταθεί τπ αντίστοιχο περιβάλλον.

Τα workflows χρησιμοποιούν εξωτερικές υπηρεσίες. Ο χρήστης πρέπει να έχει πρόσβαση σε αυτές πριν επιχειρήσει να τα τρέξει.

# Ollama:

Χρησιμοποιείται για τοπικά γλωσσικά μοντέλα και embeddings.

Ο χρήστης πρέπει:

- να έχει εγκατεστημένο Ollama,
- να έχει κατεβάσει τα απαραίτητα μοντέλα,
- να ελέγξει ότι το Ollama τρέχει κανονικά,
- να βεβαιωθεί ότι το n8n μπορεί να επικοινωνήσει με το Ollama.

# Supabase:

Χρησιμοποιείται ως βάση δεδομένων και vector database.

Ο χρήστης πρέπει:

- να έχει Supabase project,
- να έχει ενεργοποιημένο το pgvector,
- να έχει δημιουργήσει τους σωστούς πίνακες,
- να έχει δημιουργήσει τις απαραίτητες SQL functions,
- να έχει το σωστό Supabase URL,
- να έχει το σωστό Supabase API key.

# Cohere Reranker

Χρησιμοποιείται για να κάνει reranking στα αποτελέσματα που επιστρέφει το retrieval σύστημα.

Ο χρήστης πρέπει:

- να έχει Cohere account,
- να έχει API key,
- να το προσθέσει ως credential στο n8n.

# Google Drive:

Χρησιμοποιείται για ανάγνωση ή δημιουργία αρχείων.

Ο χρήστης πρέπει:

- να έχει Google account,
- να δώσει άδεια στο n8n,
- να έχει πρόσβαση στους φακέλους και στα αρχεία που χρησιμοποιούνται στα workflows.

# Google Sheets:

Χρησιμοποιείται για ανάγνωση ερωτήσεων, αποθήκευση αποτελεσμάτων ή αξιολόγηση.

Ο χρήστης πρέπει:

- να έχει πρόσβαση στα σωστά Google Sheets,
- να δώσει άδεια στο n8n,
- να ελέγξει ότι τα Sheet IDs και τα ονόματα των φύλλων είναι σωστά.

Τα credentials είναι τα στοιχεία σύνδεσης που επιτρέπουν στο n8n να επικοινωνεί με άλλες υπηρεσίες.

Για παράδειγμα:

- API key για Supabase,
- API key για Cohere,
- Google account για Google Drive,
- Google account για Google Sheets,
- σύνδεση με Ollama.

Τα credentials δεν πρέπει να κοινοποιούνται δημόσια και δεν πρέπει να περιλαμβάνονται στο αρχείο JSON.

Μετά το import, ο χρήστης πρέπει να ανοίξει τα nodes που χρειάζονται credentials και να επιλέξει ή να δημιουργήσει τα σωστά credentials μέσα στο n8n.

# Import json αρχείου στο n8n:

Αφού εγκατασταθεί και ανοίξει το n8n, ο χρήστης πρέπει να κάνει τα εξής:

1. Να ανοίξει το n8n στο browser.
2. Να πάει στη σελίδα των workflows.
3. Να επιλέξει import workflow.
4. Να επιλέξει το αρχείο:

n8n_workflows.json

5. Να περιμένει μέχρι να φορτωθούν τα workflows.
6. Να αποθηκεύσει τα workflows στο νέο n8n instance.
7. Να ανοίξει κάθε workflow ξεχωριστά και να ελέγξει τα nodes.

Μετά το import, τα workflows θα εμφανιστούν στο n8n, αλλά πιθανότατα δεν θα μπορούν να τρέξουν ακόμη, επειδή θα λείπουν credentials ή ρυθμίσεις.

# Διαδικασία μετά το import

Μετά το import, ο χρήστης πρέπει να ελέγξει προσεκτικά κάθε workflow.

Πρέπει να ελεγχθούν:

- τα credentials σε κάθε node,
- τα URLs,
- τα Google Sheet IDs,
- τα Google Drive folder IDs,
- τα ονόματα των Supabase tables,
- τα ονόματα των SQL functions,
- τα ονόματα των Ollama models,
- τα API keys,
- τα webhook URLs,
- οι συνδέσεις ανάμεσα στα nodes,
- τα nodes που καλούν άλλα workflows.

Αν κάποιο από αυτά είναι λάθος, το workflow μπορεί να αποτύχει.

Για να τρέξει ένα workflow:

1. Άνοιξε το workflow στο n8n.
2. Έλεγξε ότι όλα τα nodes είναι σωστά ρυθμισμένα.
3. Έλεγξε ότι όλα τα credentials έχουν συνδεθεί.
4. Πάτησε `Execute workflow`.
5. Παρακολούθησε αν κάθε node εκτελείται με επιτυχία.
6. Αν κάποιο node βγάλει σφάλμα, άνοιξέ το και διάβασε το error message.

Καλό είναι ο χρήστης να μην ενεργοποιήσει αμέσως όλα τα workflows. Πρώτα πρέπει να τα τρέξει δοκιμαστικά ένα-ένα.

# Προτεινόμενη σειρά εκτέλεσης

Αν τα workflows ανήκουν σε ένα σύστημα RAG chatbot, η γενική σειρά εκτέλεσης είναι η εξής:

# 1: Workflow συλλογής δεδομένων

Αυτό το workflow παίρνει δεδομένα από αρχεία, ιστοσελίδες, Google Drive ή άλλες πηγές.

# 2: Workflow επεξεργασίας

Αυτό καθαρίζει το κείμενο, το χωρίζει σε chunks και το προετοιμάζει για αποθήκευση.

# 3: Workflow αποθήκευσης και embeddings

Αυτό δημιουργεί embeddings και αποθηκεύει τα δεδομένα στη Supabase vector database.

# 4: Workflow chatbot

Αυτό είναι το workflow που δέχεται ερώτηση, ανακτά πληροφορία από τη βάση και παράγει απάντηση με χρήση LLM.

# 5: Workflow αξιολόγησης

Αυτά χρησιμοποιούνται για να αξιολογηθεί η ποιότητα των απαντήσεων, συνήθως με ερωτήσεις από Google Sheets και μετρικές αξιολόγησης.

## Συχνά προβλήματα

# Missing credentials

Αν εμφανιστεί μήνυμα ότι λείπουν credentials, ο χρήστης πρέπει να ανοίξει το αντίστοιχο node και να επιλέξει ή να δημιουργήσει νέο credential.

# Ollama connection error

Αν το n8n δεν μπορεί να συνδεθεί με το Ollama, πρέπει να ελεγχθεί αν το Ollama τρέχει και αν το URL είναι σωστό.

# Supabase error

Αν υπάρχει σφάλμα στη Supabase, πρέπει να ελεγχθούν:

- το Supabase URL,
- το API key,
- το όνομα του πίνακα,
- η ύπαρξη του pgvector,
- οι SQL functions,
- οι διαστάσεις των embeddings.

# Google Drive ή Google Sheets permission error

Αν εμφανιστεί σφάλμα δικαιωμάτων, πρέπει να ελεγχθεί αν το Google account που έχει συνδεθεί στο n8n έχει πρόσβαση στα συγκεκριμένα αρχεία ή φύλλα.

# Webhook ή Chat Trigger δεν λειτουργεί

Πρέπει να ελεγχθεί αν το workflow είναι ενεργό, αν το URL είναι σωστό και αν χρησιμοποιείται το σωστό περιβάλλον n8n.

# Τι δεν περιλαμβάνει το JSON αρχείο;

Το αρχείο `n8n_workflows.json` δεν περιλαμβάνει απαραίτητα:

- API keys,
- OAuth tokens,
- προσωπικά credentials,
- δεδομένα από τη βάση,
- αρχεία από Google Drive,
- περιεχόμενο Google Sheets,
- μοντέλα Ollama,
- ρυθμίσεις server,
- Supabase database schema,
- private URLs.

Επομένως, το import του JSON δεν αρκεί από μόνο του για πλήρη αναπαραγωγή του συστήματος.

# Τι πρέπει να ζητήσει κάποιος από τον δημιουργό των workflows;

Αν κάποιος τρίτος θέλει να τρέξει τα workflows, θα χρειαστεί επιπλέον πληροφορίες από τον δημιουργό, όπως:

- ποια workflows πρέπει να τρέξουν πρώτα,
- ποια μοντέλα Ollama χρησιμοποιούνται,
- ποιο είναι το Supabase schema,
- ποια tables χρειάζονται,
- ποιες SQL functions χρειάζονται,
- ποια Google Sheets χρησιμοποιούνται,
- ποια Google Drive folders χρησιμοποιούνται,
- ποια credentials πρέπει να δημιουργηθούν,
- ποια environment variables απαιτούνται.

Χωρίς αυτές τις πληροφορίες, τα workflows μπορούν να χρησιμοποιηθούν κυρίως ως τεκμηρίωση της αρχιτεκτονικής και όχι ως πλήρως εκτελέσιμο σύστημα.
